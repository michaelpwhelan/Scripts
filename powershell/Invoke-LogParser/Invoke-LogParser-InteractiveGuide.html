1. Getting Started

Launching Interactive Mode

Pass the -Interactive switch when invoking the script:

# Parse a log file and enter interactive mode
.\Invoke-LogParser.ps1 -Path .\firewall.log -Interactive

# Parse multiple files
.\Invoke-LogParser.ps1 -Path .\fw1.log, .\fw2.log -Interactive

# Parse all logs in a directory
.\Invoke-LogParser.ps1 -Path .\logs\ -Interactive

On launch you'll see a summary followed by the interactive prompt:

Invoke-LogParser v7.0
Loaded 4,312 events from 2 file(s)
Type 'help' for commands.

ILP [4312]>

Getting Help

Type help for a categorized command overview, or help <command> for detailed usage and examples for any specific command.

ILP [4312]> help correlate
Usage: correlate <field>
Pivot: find ALL events in the full dataset sharing field values with current results.
Example: Filter to action:deny, then correlate srcip to see everything those IPs did.

3. Query Syntax Reference

Type a query expression directly at the prompt to filter events. The result set replaces the current view, and you can undo to go back.

SyntaxDescriptionExample
field:valueExact matchaction:deny
field:val*Wildcard matchsrcip:10.0.*
field:>N / field:<NNumeric comparisonsentbyte:>1000000
NOT field:valueNegationNOT severity:Info
expr AND exprBoolean ANDaction:deny AND srcip:10.0.1.5
expr OR exprBoolean ORseverity:Critical OR severity:High

Pipe Operators

Append a pipe operator to any query to aggregate or limit results:

OperatorDescriptionExample
| countCount matching eventsaction:deny | count
| count by FGroup and count by fieldaction:deny | count by srcip
| top NTop N resultsseverity:High | top 10
| sort F ascSort by fieldseverity:High | sort Timestamp asc
| head NFirst N resultsaction:deny | head 5
| tail NLast N resultsaction:deny | tail 5

4. Inspection Commands

These commands let you examine individual events and navigate through result sets.

show <N> — Full Event Detail

Displays every field (sorted), metadata (SourceFile, LineNumber, SourceFormat), and the raw log line for event #N in the current result set.

ILP (action:deny) [234]> show 1
--- Event #1 ----------------------------------------
Timestamp     2025-03-18 14:22:31.000
Severity      High
Source         FortiGate-200F
Message       Deny UDP 10.0.1.5->8.8.8.8:53
action        deny
dstip         8.8.8.8
srcip         10.0.1.5
SourceFile    firewall.log
SourceFormat  FortiGateKV
---------------------------------------------------

raw <N> — Raw Log Line

Print the original, unparsed log line for event #N. Useful for copying to other tools or verifying field extraction.

ILP [4312]> raw 1
date=2025-03-18 time=14:22:31 devname="FortiGate-200F" ...

context <N> [lines] — Surrounding Lines

Show raw lines before and after event #N from the original file. Default context is 3 lines. Requires that the source file is accessible.

ILP [4312]> context 5 2
--- Context: 2 lines before/after ----------
  [raw] date=2025-03-18 time=14:22:29 ...
  [raw] date=2025-03-18 time=14:22:30 ...
  >>> 2025-03-18 14:22:31  HIGH  Deny UDP ... <<<
  [raw] date=2025-03-18 time=14:22:32 ...
  [raw] date=2025-03-18 time=14:22:33 ...
--------------------------------------------

next / prev — Paginated Browsing

Page through large result sets. Page size auto-detects from terminal height. Shows an "Events X-Y of Z" header on each page.

ILP (action:deny) [234]> next
Events 21-40 of 234
TIMESTAMP             SEV     SOURCE           MESSAGE
2025-03-18 14:23:01   High    FortiGate-200F   Deny TCP ...

goto <N> — Jump to Page

Jump directly to the page containing event #N.

ILP (action:deny) [234]> goto 100
Events 91-110 of 234
...

diff <N> <M> — Compare Two Events

Side-by-side comparison of two events, showing only fields that differ. Useful for spotting what changed between similar events.

ILP [4312]> diff 1 2
--- Diff: #1 vs #2 ----------------------------
FIELD               #1 VALUE                       #2 VALUE
Timestamp           2025-03-18 14:22:31            2025-03-18 14:22:45
srcip               10.0.1.5                       10.0.1.8
dstport             53                             443

5. Investigation Commands

These commands help you explore patterns, pivot across data, and build an investigative narrative.

whois <field> / inspect <field> — Field Value Summary

Collect unique values for a field with counts, displayed as a bar chart (top 20). Both command names are equivalent.

ILP (action:deny) [234]> whois srcip
--- srcip (18 unique) ----------------------------
10.0.1.5              ████████████████████  87
10.0.2.12             ███████████           42
192.168.1.100         ██████                28
...

timeline — ASCII Histogram

Shows event distribution over time. Automatically picks bucket intervals based on the time span: <2 hours = 5-minute buckets, <48 hours = 1-hour buckets, otherwise daily buckets.

ILP [4312]> timeline
--- Timeline ----------------------------------------
03-18 08:00   ████████                      24
03-18 09:00   ██████████████████████████     78
03-18 10:00   ████████████████               48
03-18 11:00   ████████████████████           61
...

correlate <field> — Pivot on Field Values

The key investigative command. Collects all unique values of <field> from the current result set, then finds all events in the full dataset that share those values. Saves previous state for undo.

Investigation pattern: Filter to suspicious events (e.g., action:deny), then correlate srcip to see everything those source IPs did — including allowed connections, login attempts, and other activity. 
ILP [4312]> action:deny AND severity:High
87 events

ILP (action:deny AND severity:High) [87]> correlate srcip
Correlated on 12 unique srcip values: 1,247 events

ILP (correlate:srcip) [1247]> whois action
--- action (4 unique) ----------------------------
allow                 ████████████████████  892
deny                  ████████              312
dns                   ████                  38
ipsec                 █                     5

search <text> — Free-Text Search

Case-insensitive substring search across Severity, Source, Message, RawLine, and all Extra field values. Narrows the current result set.

ILP [4312]> search admin
156 events matching 'admin'

exclude <field>:<value> — Exclude Matching Events

Remove events matching a field:value pattern from the current result set. Supports wildcards. Saves previous state for undo.

ILP (correlate:srcip) [1247]> exclude severity:Info
892 events (excluded severity=Info)

ILP (exclude severity:Info) [892]> exclude srcip:10.0.1.*
605 events (excluded srcip=10.0.1.*)

unique <field> — Deduplicate by Field

Keep only the first event per unique field value. Essential for “which distinct IPs/users did X” investigations.

ILP (correlate:srcip) [1247]> unique srcip
12 unique srcip values (from 1,247 events)

highlight <pattern> / unhighlight

Set a text pattern to highlight in yellow in all subsequent output (show, raw, grid, table, and list views). Running highlight with the same pattern again toggles it off. Use unhighlight to clear explicitly.

ILP [4312]> highlight deny
Highlighting: deny

ILP [4312] H> unhighlight
Highlight cleared.

Time-Based Investigation

anchor <N> <duration> [before|after]

Find all events within a time window around event #N. Durations use suffixes: s=seconds, m=minutes, h=hours, d=days.

ILP [4312]> anchor 42 5m
187 events within 5m of event #42

ILP [4312]> anchor 42 30s after
12 events within 30s after event #42

between <start> to <end>

Filter to an explicit time window. Supports flexible timestamp formats: HH:mm, MM-dd HH:mm, yyyy-MM-dd HH:mm:ss.

ILP [4312]> between 14:00 to 15:00
342 events between 2026-03-18 14:00:00 and 2026-03-18 15:00:00

window <duration> / since <duration>

Show events from the last N seconds/minutes/hours/days of the dataset.

ILP [4312]> window 1h
891 events in last 1h

6. Analysis & Display Commands

stats — Full Statistics

Displays a comprehensive statistical summary including severity distribution, top source IPs, time range, and per-file breakdown.

top-talkers — Quick Summary

Renders four bar-chart sections: top source IPs, destination IPs, sources, and event IDs. A fast way to spot dominant actors.

ILP [4312]> top-talkers
--- Top Source IPs ---------------------------
10.0.1.5              ████████████████████  412
10.0.2.12             ███████████           287
...
--- Top Dest IPs ----------------------------
8.8.8.8               ████████████████████  198
1.1.1.1               ██████████████        142
...

severity — Severity Breakdown

Bar chart showing Critical, High, Medium, Low, and Info counts with percentages. Severity labels are color-coded.

ILP [4312]> severity
--- Severity Breakdown --------------------------
CRITICAL   ██                            42 (1.0%)
HIGH       ████████                     312 (7.2%)
MEDIUM     ██████████████              1,024 (23.7%)
LOW        ████████████████████        1,891 (43.8%)
INFO       ██████████████              1,043 (24.2%)

heatmap — 24-Hour Event Density

Vertical bar grid showing events per hour (00:00–23:00). Hours where >30% of events are Critical or High severity are shown in red; otherwise cyan.

ILP [4312]> heatmap
--- 24-Hour Heatmap -----------------------------
00:00  ████                     12
01:00  ██                        8
...
09:00  ████████████████████    198
10:00  ██████████████████      167
...

analyze <engine> — Run Analysis

Run a built-in analysis engine on the current result set:

  • analyze failed-logins — Failed login pattern analysis
  • analyze vpn-sessions — VPN session reconstruction
  • analyze ipsec-tunnel — IPsec tunnel status analysis

report <type> [--export <path>] — Generate Report

Generate and optionally export a report:

  • report summary / report morning — Summary or detailed morning report
  • report audit / report compliance — Audit or compliance report
  • report timeline — Chronological timeline report
ILP [4312]> report summary --export /tmp/report.html

Enrichment & Intelligence

explain <value>

Context-aware lookup against embedded enrichment data: Windows Event IDs, NPS reason codes, FortiGate log ID ranges, Kerberos encryption types, and FortiGate subtypes.

ILP [4312]> explain 4625
Windows Event ID 4625: Failed logon
  Significance: Brute force indicator when clustered.

ILP [4312]> explain 0x17
Kerberos Encryption Type 0x17: RC4-HMAC-MD5
  Significance: Legacy RC4-HMAC. Consider AES migration.

ioc import <path> / ioc match / ioc export <path>

Import IOC values from a text/CSV file, scan events against them, and export matches.

ILP [4312]> ioc import /tmp/iocs.txt
Loaded 47 IOCs as $_ioc_iocs.txt

ILP [4312]> ioc match
23 events matched 8/47 IOCs

policy <id> / policy-stats

Look up FortiGate policy details from event data, or show event counts per policy ID.

7. Data Management Commands

CommandDescription
load <file>Load an additional log file into the session. Format is auto-detected. Events are merged and sorted by timestamp.
sourcesList all loaded log sources with their detected format and event count.
fieldsList all available field names across all loaded events, including Extra fields.
export <fmt> <path>Export current filtered events. Formats: csv, json, html.
clearClear the active filter and restore the full event set. Previous state is saved for undo. Pinned events are preserved.
undoRestore the previous filter state. Works like a browser back button — each filter or clear operation pushes to the undo stack.
ILP [4312]> action:deny
234 events

ILP (action:deny) [234]> undo
Restored. 4,312 events

ILP [4312]> export csv /tmp/all-events.csv

8. Session Commands

history / !N

Show the last 30 commands entered during this session. Use !N to re-execute command #N from history.

ILP [4312]> history
  1. action:deny AND severity:High
  2. whois srcip
  3. correlate srcip

ILP [4312]> !1
>> action:deny AND severity:High
87 events

alias [name] [query]

Create query shortcuts. Without arguments, lists all defined aliases. Aliases are expanded before command processing, so they work with any command syntax.

ILP [4312]> alias denied action:deny
Alias 'denied' -> 'action:deny'

ILP [4312]> denied
234 events

ILP [4312]> alias
  denied -> action:deny

pin / unpin / pinned

Pin saves the current filtered event set. Pinned events persist through clear operations and are indicated in the prompt. Use pinned to switch to viewing pinned events, and undo to return. Use unpin to release.

ILP (action:deny) [234]> pin
Pinned 234 events.

ILP (action:deny) [234] [pin:234]> clear
Filter cleared.
(234 pinned events still available)

ILP [4312] [pin:234]> pinned
Switched to 234 pinned events

ILP (pinned) [234] [pin:234]> undo
Restored. 4,312 events

output <format>

Change the display format for event listings. Options:

  • Grid (default) — Compact columnar display
  • Table — Box-drawn table with borders
  • List — One line per event, minimal formatting

columns <field1,field2,...> / columns reset

Set which fields to display in Grid, Table, and List output. Use fields to see available field names. Use columns reset to restore defaults.

ILP [4312]> columns Timestamp,srcip,dstip,action,Message
Display columns: Timestamp, srcip, dstip, action, Message

ILP [4312]> columns reset
Columns reset to default.

width <field> <N>

Set a custom column width for a specific field in Grid and Table output. Persists for the session.

ILP [4312]> width Source 30
Column width 'Source' set to 30

Investigation Journal & Bookmarks

The journal automatically logs every command and its result. Add manual notes with note, tag individual events with bookmark.

note "<text>"

Add an analyst note to the journal.

ILP [4312]> note "user jsmith appears to be source of lateral movement"
Note added.

journal / journal export <path>

View the investigation timeline, or export as markdown for incident reports.

bookmark <N> ["note"] / bookmarks

Tag individual events with notes. bookmarks lists them. bookmarks show displays full detail. bookmarks export <path> to CSV.

ILP [4312]> bookmark 42 "initial compromise indicator"
Bookmarked event #42: initial compromise indicator

Watchlist

Define field values to flag across all output. Watched values are marked in show, whois, map, and grid output.

ILP [4312]> watch srcip 10.0.1.5,10.0.1.8
Watching srcip: 10.0.1.5, 10.0.1.8

ILP [4312] [W]> watch remove srcip 10.0.1.8
ILP [4312] [W]> watch clear
Watchlists can be persisted via .ilp-config.json or session save.

Variables

Capture field values for reuse across queries and correlations.

ILP (action:deny) [87]> $badips = select srcip
$badips = 12 unique srcip values (first 5: 10.0.1.5, 10.0.1.8, ...)

ILP [4312]> srcip:$badips
1,247 events

ILP [4312]> correlate srcip using $badips
Correlated on 12 unique srcip values: 1,247 events

ILP [4312]> vars
  $badips (12 values)

Compliance & Evidence Collection

Structured evidence collection for FFIEC compliance exams.

ILP [4312]> evidence start "AC-2: Account Management"
Evidence collection started for: AC-2: Account Management

ILP [4312] [E:AC-2]> EventID:4720 OR EventID:4726
47 events

ILP (...) [47] [E:AC-2]> evidence capture "Account creation and deletion"
Captured: Account creation and deletion (47 events)

ILP [4312] [E:AC-2]> evidence finish
ILP [4312]> evidence export /tmp/ac2-evidence.html

Playbooks

Repeatable investigation workflows stored as .ilp-playbook files. Pause points allow analyst review between sections.

ILP [4312]> playbook list
  daily-review    Standard morning log review  (12 commands)

ILP [4312]> playbook run daily-review
ILP [4312]> playbook create my-investigation

Session Persistence

Save and restore session state across multiple sessions. Aliases, watchlists, bookmarks, command history, and journal are preserved.

ILP [4312]> session save /tmp/investigation.ilp-session
Session saved.

# Later, after reloading the same log files:
ILP [4312]> session restore /tmp/investigation.ilp-session
Session restored (3 aliases, 2 watchlists)

Investigation Workflows

Incident Response Walkthrough

Scenario: An alert fired for denied traffic from an internal IP. You need to determine the scope of the activity.

# 1. Launch with the firewall log
.\Invoke-LogParser.ps1 -Path .\fw.log -Interactive

# 2. Find the denied events
ILP [4312]> action:deny AND severity:High
87 events

# 3. Who's generating these?
ILP (action:deny AND severity:High) [87]> whois srcip
--- srcip (3 unique) ---
10.0.1.5    ████████████████████  52
10.0.1.8    ██████████            28
10.0.2.3    ███                    7

# 4. See EVERYTHING those IPs did (the pivot)
ILP (...) [87]> correlate srcip
Correlated on 3 unique srcip values: 1,247 events

# 5. What actions did they take?
ILP (correlate:srcip) [1247]> whois action
allow  ████████████████████  892
deny   ████████              312
dns    ███                    38
ipsec  █                       5

# 6. Focus the display on relevant fields
ILP (correlate:srcip) [1247]> columns srcip,dstip,action,dstport,Message

# 7. Remove noise
ILP (correlate:srcip) [1247]> exclude severity:Info
892 events (excluded severity=Info)

# 8. Look at the timeline
ILP (exclude severity:Info) [892]> timeline

# 9. Inspect a suspicious event
ILP (...) [892]> show 1

# 10. Compare two events
ILP (...) [892]> diff 1 15

# 11. Pin results and continue investigating
ILP (...) [892]> pin

# 12. Export for the incident report
ILP (...) [892]> report summary --export /tmp/incident.html

Daily Log Review

# Quick overview
ILP [4312]> severity
ILP [4312]> heatmap
ILP [4312]> top-talkers

# Drill into critical events
ILP [4312]> severity:Critical
ILP (severity:Critical) [42]> whois Source
ILP (severity:Critical) [42]> show 1

ILP [4312]> analyze failed-logins

# Generate morning report
ILP [4312]> report morning --export /tmp/morning.html
