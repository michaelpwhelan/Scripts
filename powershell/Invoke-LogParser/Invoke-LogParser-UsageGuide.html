1. Quick Start

Drop Invoke-LogParser.ps1 anywhere on your system. No installation or modules required.

# Parse and display with auto-detection
.\Invoke-LogParser.ps1 -Path firewall.log

# Filter denied traffic, compact grid
.\Invoke-LogParser.ps1 -Path firewall.log -Filter "action:deny" -OutputFormat Grid

# Quick digest of all logs in a directory
.\Invoke-LogParser.ps1 -Path *.log -StatsOnly

# Multi-file parsing across sources
.\Invoke-LogParser.ps1 -Path firewall.log,switch.log,dc01-security.xml

# Extract and parse a zip archive
.\Invoke-LogParser.ps1 -Path logs-export.zip

# Analyze failed logins across sources
.\Invoke-LogParser.ps1 -Path firewall.log,dc01-security.xml -Analyze FailedLogins

# Generate an HTML report and open it
.\Invoke-LogParser.ps1 -Path firewall.log -Report Summary -ExportPath report.html -ExportFormat Html -Open

2. Requirements

  • PowerShell 5.1+ — Windows PowerShell or pwsh 7+
  • No external modules — pure PowerShell + .NET Framework
  • Single file — just Invoke-LogParser.ps1, no helper files needed
Binary .evtx files require Windows with Get-WinEvent. For cross-platform use, export Windows Event Logs to XML first.

3. Supported Formats

ParserDescriptionAuto-Detect
FortiGateConf FortiGate configuration file (.conf) #config-version=FG or config system global
FortiGateKV Fortinet key=value syslog (FortiGate, FortiClient EMS, FortiAP) logid= + type= + devname=/devid=
FortiClientLocal FortiClient local log (tab-delimited debug + structured events) Tab-delimited timestamp, fctver=, devid=FCT, or legacy bracket format
FortiSwitchEvent FortiSwitch event log (port, STP, MAC, 802.1X) devid=FS or devtype="FortiSwitch"
WindowsEvtx Windows Event Log (binary .evtx or XML export) .evtx extension or <Event xmlns=

Compressed .zip archives are automatically extracted and parsed. Format is auto-detected from file content — use -Format to override.

.\Invoke-LogParser.ps1 -ListParsers    # show all parsers

FortiClient Local Log Format

FortiClient local logs (exported from C:\Program Files\Fortinet\FortiClient\logs\) contain two interleaved line types:

  • Debug lines — TIMESTAMP<TAB>LEVEL<TAB>CATEGORY<TAB>MESSAGE with internal diagnostic messages. Parsed with LogType=Debug.
  • Structured events — same tab-delimited layout, but the message field contains a full Fortinet key=value payload (date=... time=... type=... msg="..."). All key=value pairs are expanded into properties. Parsed with LogType=Event.
# Skip debug noise, show only structured events
.\Invoke-LogParser.ps1 -Path forticlient.log -Filter "LogType:Event"

# Find compliance failures
.\Invoke-LogParser.ps1 -Path forticlient.log -Filter "LogType:Event AND eventtype:compliance"

The legacy bracket format ([YYYY-MM-DD HH:MM:SS] [level] [module] message) from older FortiClient versions is also supported.

4. Output Formats

FormatFlagDescription
Table-OutputFormat TableBox-drawing bordered table (default)
Grid-OutputFormat GridCompact space-padded columns
List-OutputFormat ListDetailed field-per-line for each event
Raw-OutputFormat RawPSObjects for pipeline (Where-Object, Export-Csv, etc.)
Json-OutputFormat JsonJSON Lines (one object per line)
Csv-OutputFormat CsvCSV with auto-discovered headers

All output formats respect the -Fields parameter to select specific columns. When output is piped, color is auto-disabled.

# Select specific fields across any format
.\Invoke-LogParser.ps1 -Path firewall.log -Fields "Timestamp,srcip,dstip,action" -OutputFormat Csv

# Pipeline integration with Raw output
.\Invoke-LogParser.ps1 -Path firewall.log -OutputFormat Raw | Where-Object { $_.Extra.dstport -eq '3389' }

5. Query Syntax

Used with -Filter and in interactive mode. Supports hyphenated field names (e.g., ips-sensor:default, utm-status:enable).

Field Matching

SyntaxDescriptionExample
field:valueExact match (case-insensitive)action:deny
field:val*Wildcard matchsrcip:10.12.*
field:>NNumeric greater-thandstport:>1024
field:<NNumeric less-thansentbyte:<100
NOT field:valueNegationNOT action:accept
expr AND exprBoolean ANDaction:deny AND srcip:10.*
expr OR exprBoolean ORseverity:High OR severity:Critical

Pipeline Operators

Append pipeline operators after a filter using |:

OperatorDescriptionExample
| countCount matching resultsaction:deny | count
| count by FGroup and count by fieldaction:deny | count by dstport
| top NTop N resultsaction:deny | top 10
| sort F [asc|desc]Sort by fieldseverity:* | sort Timestamp desc
| head NFirst N events| head 50
| tail NLast N events| tail 20
| timeline IntervalTime-bucketed histogram| timeline 1h
| table F1,F2,...Select display columns| table srcip,dstip,action

6. Analysis Engines

.\Invoke-LogParser.ps1 -Path <files> -Analyze <engine>
.\Invoke-LogParser.ps1 -ListAnalyzers   # show all engines
EngineDescription
FailedLogins Cross-source failed authentication aggregation. Detects Windows Event IDs 4625, 4771, 6273; NPS Access-Reject; FortiGate user auth denials. Aggregates by user with count, unique source IPs, first/last seen, and source type.
VpnSessions VPN session lifecycle tracking. Maps tunnel-up/down, ssl-new-con, login/logout events into session objects with duration. Detects impossible travel: same user from different /16 public IP ranges within 30 minutes. Private (RFC 1918) and IPv6 addresses are excluded from travel checks.
IpsecTunnel Per-tunnel health metrics. Tracks up/down events, negotiation success/failure, DPD timeouts. Calculates flap count (state changes within 5 minutes) and uptime percentage.
Summary General statistics: total events, severity distribution, top source IPs, top event IDs, events per hour. 
# Analyze failed logins across firewall and domain controller logs
.\Invoke-LogParser.ps1 -Path firewall.log,dc01-security.xml -Analyze FailedLogins

# Check VPN session health and impossible travel
.\Invoke-LogParser.ps1 -Path vpn.log -Analyze VpnSessions

# Monitor IPsec tunnel stability
.\Invoke-LogParser.ps1 -Path firewall.log -Filter "subtype:ipsecvpn" -Analyze IpsecTunnel

8. FortiGate Config Tools

Section Extraction

# Extract a config section
.\Invoke-LogParser.ps1 -Path config.conf -Section "firewall policy"

# Extract with wildcard
.\Invoke-LogParser.ps1 -Path config.conf -Section "firewall*"

# Extract system interfaces
.\Invoke-LogParser.ps1 -Path config.conf -Section "system interface"

Security Flagging

When parsing FortiGate configuration files, the config parser automatically flags security issues on each policy/setting:

FlagCondition
INSECURE-MGMTHTTP or Telnet enabled on an interface's allowaccess
NO-IPSAccept policy with no IPS sensor assigned
NO-UTMAccept policy with no UTM profiles (AV, web filter, app control, etc.)
NO-LOGGINGPolicy with logging disabled or defaulting to off
PERMISSIVEPolicy with srcaddr=all, dstaddr=all, service=ALL
WEAK-PASSWD-POLICYPassword policy disabled or minimum length < 8
WEAK-SSL-INSPECTCertificate-inspection mode (no deep inspection)
For config diffing, use the dedicated Compare-FortiGateConfigs.ps1 script in this repo, which provides LCS-based section-by-section comparison.

10. Live Tail Mode

Monitor a log file in real-time with parsed, color-coded output:

.\Invoke-LogParser.ps1 -Path firewall.log -Tail
.\Invoke-LogParser.ps1 -Path firewall.log -Tail -TailLines 50
.\Invoke-LogParser.ps1 -Path firewall.log -Tail -Filter "action:deny"
.\Invoke-LogParser.ps1 -Path firewall.log -Tail -Highlight "10\.12\.1\.50"

Uses Get-Content -Wait under the hood. Press Ctrl+C to stop.

11. Enrichment Data

All enrichment data is embedded in the script — no external data files needed:

  • Windows Event IDs — 100+ annotated IDs covering Security, AD, NPS, Hyper-V, DHCP, and Defender
  • NPS/RADIUS reason codes — 40+ translations for denial reasons
  • FortiGate mappings — type/subtype descriptions, log ID ranges, FortiClient module names
  • Severity normalization — maps vendor-specific levels to a standard five-level scale: Critical, High, Medium, Low, Info

12. Parameter Reference

Input

ParameterDescription
-PathLog file(s), glob patterns, or .zip archives. Accepts multiple comma-separated values.
-FormatForce a specific parser: FortiGateConf, FortiGateKV, FortiClientLocal, FortiSwitchEvent, WindowsEvtx. Omit for auto-detection.
-InputObjectAccept piped input (requires -Format since auto-detection needs a file).

Filtering

ParameterDescription
-FilterQuery string using field matching, boolean operators, and pipeline ops.
-RegexRegex filter matched against raw log lines. Invalid patterns produce a warning.
-SurroundShow all events within N seconds of each matching event.

Output

ParameterDescription
-OutputFormatDisplay format: Table (default), Grid, List, Raw, Json, Csv.
-FieldsComma-separated fields to display. Works with all output formats.
-MaxResultsLimit the number of displayed events.
-HighlightColor-highlight text matching this pattern without filtering events out.
-ContextShow N raw log lines before and after each matching event (like grep -C).
-NoColorDisable colored output.
-QuietSuppress console output (useful with -ExportPath).
-ClipboardCopy output to clipboard (via Set-Clipboard).

Analysis & Reports

ParameterDescription
-AnalyzeRun an analysis engine: FailedLogins, VpnSessions, IpsecTunnel, Summary.
-ReportGenerate a report: Summary, Morning, Audit, Compliance, Timeline.
-ExportPathFile path to write a report.
-ExportFormatExport format: Html, Csv, Json.
-OpenAuto-open the exported file after creation.
-StatsOnlyShow only parse statistics / quick digest, not individual events.

Modes

ParameterDescription
-InteractiveLaunch interactive query mode (REPL).
-TailMonitor the log file in real-time.
-TailLinesNumber of existing lines to show before tailing. Default: 20.
-SectionExtract a specific section from a FortiGate config file. Supports wildcards.

Discovery

ParameterDescription
-ListParsersList available log format parsers.
-ListAnalyzersList available analysis engines.
-ListReportsList available report types.

13. Architecture

Single file (Invoke-LogParser.ps1, ~2,900 lines) organized into 13 sections:

#SectionPurpose
1Parameters & ConfigurationCmdletBinding, ANSI color system, script-level state
2Enrichment DataFortiGate mappings, Windows Event IDs, NPS codes
3Parsers5 format parsers + auto-detect + dispatcher
4Filter EngineQuery parser, field matching, pipeline operators
5Analysis EnginesFailed logins, VPN sessions, IPsec tunnel health
6Output FormatterTable, Grid, List, Raw, Json, Csv + analysis result renderers
7Config ToolsFortiGate section extraction, syntax highlighting
8Report EngineReport generators + HTML/console renderers
9Interactive ModeREPL with query, analyze, export commands
10Live Tail ModeReal-time parsed output via Get-Content -Wait
11Output HelpersClipboard, file open
12Zip ExtractionArchive handling with nested zip support and format discovery
13Main ExecutionOrchestration, pipeline support, cleanup

14. Workflow Examples

Daily Security Review

# 1. Quick stats across all log sources
.\Invoke-LogParser.ps1 -Path *.log,dc01-security.xml -StatsOnly

# 2. Morning briefing report
.\Invoke-LogParser.ps1 -Path *.log,dc01-security.xml -Report Morning

# 3. Drill into failed logins
.\Invoke-LogParser.ps1 -Path *.log,dc01-security.xml -Analyze FailedLogins

# 4. Investigate a specific user
.\Invoke-LogParser.ps1 -Path *.log,dc01-security.xml -Filter "user:jsmith" -OutputFormat Grid

Incident Investigation

# 1. Find critical events
.\Invoke-LogParser.ps1 -Path firewall.log -Filter "severity:Critical" -OutputFormat Grid

# 2. Get surrounding context (30 second window)
.\Invoke-LogParser.ps1 -Path firewall.log,switch.log -Filter "severity:Critical" -Surround 30

# 3. Regex search with raw line context
.\Invoke-LogParser.ps1 -Path firewall.log -Regex "tunnel.*down|phase[12].*fail" -Context 5

# 4. Export timeline for the incident
.\Invoke-LogParser.ps1 -Path firewall.log,switch.log -Report Timeline -ExportPath incident.html -ExportFormat Html

Compliance Audit

# 1. Audit report from domain controller logs
.\Invoke-LogParser.ps1 -Path dc01-security.xml -Report Audit -ExportPath audit.html -ExportFormat Html

# 2. FFIEC compliance mapping
.\Invoke-LogParser.ps1 -Path dc01-security.xml,firewall.log -Report Compliance -ExportPath compliance.html -ExportFormat Html

# 3. Review FortiGate config for security issues
.\Invoke-LogParser.ps1 -Path config.conf -Filter "severity:Medium OR severity:High" -OutputFormat Grid

FortiClient Endpoint Logs

# 1. Parse FortiClient local log, show only structured events
.\Invoke-LogParser.ps1 -Path forticlient.log -Filter "LogType:Event" -OutputFormat Grid

# 2. Find compliance failures
.\Invoke-LogParser.ps1 -Path forticlient.log -Filter "LogType:Event AND eventtype:compliance"

# 3. Export all events from a specific endpoint
.\Invoke-LogParser.ps1 -Path forticlient.log -Filter "hostname:WS-PC01" -OutputFormat Csv > endpoint-events.csv

Pipeline Integration

# Pipe into native PowerShell for custom processing
.\Invoke-LogParser.ps1 -Path firewall.log -OutputFormat Raw |
    Where-Object { $_.Extra.dstport -eq '3389' } |
    Group-Object { $_.Extra.srcip } |
    Sort-Object Count -Descending |
    Select-Object Count, Name -First 10

# Feed piped content into the parser
Get-Content firewall.log | .\Invoke-LogParser.ps1 -Format FortiGateKV -Filter "action:deny" -OutputFormat Json
